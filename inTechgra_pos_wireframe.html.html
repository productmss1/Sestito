Story 1.1 — Proses Transaksi dengan Benefit Program

Gambaran UI

Halaman transaksi POS yang sudah ada, dengan tambahan panel kanan yang mencakup:

  • Input No HP Customer
  • Section Voucher (sisa kuota + tombol Apply Voucher)
  • Ringkasan harga (subtotal, diskon, total)
  • Selector fulfillment (Pickup / Delivery)
  • Tombol PROSES TRANSAKSI dengan floating panel

Detail Functional

1. Input No HP Customer
  • Field text input untuk No HP customer pada tombol +
  • Tidak ada API call pada saat input — No HP disimpan di state lokal
  • No HP digunakan saat validasi pada step PROSES
2. Tampilan Section Voucher

Kondisi tampil:

  • Section voucher hanya muncul jika toko memiliki flag is_100_toko = true
  • Jika flag tidak ada → section voucher tidak dirender sama sekali
  • Tampilan hanya tombol apply voucher, teks sisa kuota dan tombol apply voucher muncul

Isi section voucher:

  • Teks: Sisa Kuota Voucher: {n} — selalu ditampilkan termasuk saat nilai = 0
  • Tombol Apply Voucher:
    • kuota > 0 → tombol aktif (enabled)
    • kuota = 0 → tombol disabled (greyed out), tidak bisa diklik
3. Apply Voucher

Trigger: Kasir klik tombol Apply Voucher

Behavior:

  • Sistem langsung mengupdate tampilan harga:
    • Total = Subtotal - Rp 30.000
  • Badge/label "Voucher Rp 30.000 diterapkan ✓" muncul di UI
  • Harga sudah berubah di layar sebelum PROSES diklik
  • Kasir bisa cancel/remove voucher jika diperlukan → harga kembali ke subtotal penuh
4. Validasi saat PROSES TRANSAKSI

Trigger: Kasir klik tombol PROSES TRANSAKSI

Sistem melakukan 2 layer validasi secara berurutan via API ke server 1000saudara:

Layer 1 — Validasi Status Pelanggan 1000saudara

Request: GET /customer/lookup?phone={no_hp}

Response:
- found: true/false
- is_first_transaction: true/false

Kondisi:

  • found = false (No HP tidak terdaftar sebagai pelanggan 1000saudara):
    • Tampilkan notif error: "Pelanggan belum terdaftar di 1000saudara. Voucher tidak dapat digunakan."
    • Voucher otomatis di-remove (harga kembali ke subtotal penuh)
    • Kasir lanjutkan transaksi tanpa voucher
    • Stop — tidak lanjut ke Layer 2
  • found = true → lanjut ke Layer 2

Layer 2 — Validasi Voucher (First Transaction)

Kondisi (menggunakan data dari response Layer 1):

  • is_first_transaction = false (customer sudah pernah transaksi di 1000saudara):
    • Tampilkan notif error: "Voucher hanya berlaku untuk transaksi pertama. Voucher tidak dapat digunakan."
    • Voucher otomatis di-remove (harga kembali ke subtotal penuh)
    • Kasir lanjutkan transaksi tanpa voucher
  • is_first_transaction = true → voucher valid ✅:
    • Total tetap = Subtotal - Rp 30.000
    • Kuota voucher toko berkurang 1
    • Lanjut ke proses push cart
5. Push Cart ke 1000saudara

Setelah validasi selesai (dengan atau tanpa voucher):

Request: POST /cart/push

Body:
{
  "customer_phone": "08xx",
  "store_id": "xxx",
  "items": [...],
  "subtotal": 250000,
  "voucher_applied": true/false,
  "discount": 30000,
  "total": 220000,
  "fulfillment_type": "pickup" | "delivery"
}
  • Cart masuk ke akun customer di 1000saudara
  • Status awal di sisi POS: "Dalam Keranjang"
  • Pesanan muncul di Modul Order Management (Epic 2)

Ringkasan API — Epic 1

Method Endpoint Trigger Keterangan
GET /customer/lookup Saat PROSES diklik Cek status pelanggan & first transaction
POST /cart/push Setelah validasi berhasil Push transaksi ke cart customer
GET /voucher/quota?store_id= Saat halaman transaksi dibuka Ambil sisa kuota voucher toko

Error States — Epic 1

Kuota voucher = 0 (tidak ada pesan error — tombol disabled) Tombol Apply Voucher disabled

High Level Concept

High-Level Concept: Modul Akuisisi Phygital (inTechgra x 1000saudara)

Author: Product Manager inTechgra
Status: Draft
Target Release: Fase 1 (Program 100 Toko Anchor Jawa)

1. Latar Belakang & Konteks Bisnis (Background)

PT MSS mengelola dua produk: inTechgra (POS Toko Bangunan/B2B) dan 1000saudara (Marketplace/B2C). Saat ini, transaksi di 100 toko bangunan anchor masih berjalan secara offline konvensional.

Untuk membesarkan ekosistem, kita akan menjalankan strategi Phygital Acquisition, yaitu memanfaatkan momentum transaksi kasir fisik di inTechgra sebagai pintu masuk untuk mendaftarkan dan memindahkan data belanja pelanggan offline ke dalam platform digital 1000saudara.

2. Objektif Produk (Product Objectives)

  • Mengurangi Friction Registrasi: Membantu user mendaftar ke 1000saudara secara instan saat melakukan pembayaran di kasir toko fisik.
  • Insentif Validasi: Memberikan hook berupa potongan langsung Rp30.000 untuk transaksi pertama di platform.
  • Target Transisi (Fase 2): Mendorong pelanggan bertransisi menjadi pembeli online mandiri di 1000saudara setelah onboarding awal berhasil.

3. Anggaran & Benefit Toko (Investasi Program)

Anggaran program disetujui direksi sebesar Maks. Rp 3.3 Miliar dengan rincian:

  • Renovasi Toko Fisik: Maksimal Rp 30.000.000 per toko untuk branding.
  • Program Voucher: 100 voucher diskon Rp 30.000 per toko (ditanggung penuh PT MSS).
  • Eksklusif Fee 0%: Potongan fee transaksi POS ke 1000saudara gratis selama 6 bulan pertama, dapat diperpanjang hingga 1-2 tahun berdasarkan performa revenue.

4. Alur Pengalaman Pengguna (App First)

Pelanggan instal app di kasir fisik → kasir apply voucher Rp 30.000 → potongan harga langsung didapatkan pelanggan di kasir.

5. Aturan Bisnis & Expiry

  • Masa Kedaluwarsa Cart: 1x24 jam. Jika kedaluwarsa, voucher kembali ke kuota toko secara otomatis.
  • Jadwal Rekonsiliasi: Reimburse klaim diskon dicairkan 2x seminggu oleh Finance PT MSS.

6. Metrik Keberhasilan (Goals)

  • Fase 1: 10.000 pendaftaran pelanggan baru dan redemption rate voucher >80%.
